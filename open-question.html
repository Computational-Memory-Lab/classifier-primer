Chapter 7

The question we are trying to answer

Everything so far has been background. This is the actual open problem — the reason the project exists, and the thing a new lab member could plausibly help solve.

The observation

Almost every EEG classification study in the literature uses spectral features — how much rhythmic activity there is at each frequency — rather than the time-domain voltage we have been using. And when someone compares them head to head on the same data, spectral wins.

Danyluik, Chakravarty & Caplan (2025) did exactly that comparison, in this lab, on a learning task. Their result:

The gap this project is about
Danyluik et al. (2025), Table 3, all trials, N = 45. Same participants, same trials, same classifiers — the only thing that changes between each pair is how the EEG was represented. Bars are 95% confidence intervals.

In their own words: “Time-frequency spectral features produced better classifications (AUC ∼0.7) than time-domain features.” The difference was not subtle — t(44) = 6.38 for LDA and t(44) = 4.86 for SVM.

This is the cleanest possible version of the comparison: it is a within-participant test on one dataset. Nothing differs but the representation, so the usual escape routes — different people, different task, different preprocessing — are all closed off.

So: why? That question is the whole project. It has two very different possible answers, and they matter enormously for what the field should do next.

Two possible answers

POSSIBILITY 1

Brain rhythms really are where memory lives

Oscillations — theta, alpha — are a genuine mechanism of memory encoding and retrieval. Spectral features win because they measure the thing that actually does the work, and voltage-over-time only sees it indirectly. If this is true, the field should keep going with spectral features and think harder about oscillatory mechanisms.

POSSIBILITY 2

It is an artifact of the transformation

Computing a spectrogram is a specific mathematical operation: chop the signal into windows, work out how much of each frequency is present in each one, and keep the amount — the magnitude — while discarding the phase, which is where in its cycle each rhythm happened to be. Maybe that operation packages the same information in a form a linear classifier can use more easily. If so, the “oscillation” story is a misinterpretation of a data-processing side effect.

This is not hair-splitting. One answer says something about how the brain works; the other says something about how our analysis works. Telling them apart takes careful, unglamorous methodological work — which is what this project is.

What we have ruled out so far

Chapter 6 showed the first real attempt at possibility 2. A spectrogram passes voltage through a nonlinearity — squaring, essentially — so: does simply squaring the time-domain features reproduce the advantage?

Testing the simplest version of possibility 2
If nonlinearity alone explained it, the transformed bars would go up. They go down.

No. Squaring makes it worse — dramatically so for FA vs CR, which collapses to chance. The manuscript states it plainly: “The spectrographic advantage was not attributable to voltage being passed through a nonlinear transformation.”

And there is a clue in which transformations failed. Absolute value and squaring both discard the sign of the voltage; cube keeps it, and cube survives. So the sign carries a lot of what the classifier was using. That raises the obvious next question. A spectrogram discards phase, which is sign information of a kind — so why does it not suffer the same way?

That is roughly where the project is now. The simplest explanation is ruled out, the result is still unexplained, and the next steps are open.

What is missing, and what you could do

Here is the honest state of things, which is also the opportunity. The spectral analysis that would let us test this directly on our data is not in the repository. There is no FFT, no wavelet, no band-power extraction anywhere in the codebase. The manuscript has a table of spectral results whose generating code nobody has found.

Which means the most useful thing an incoming person could do is easy to state:

Open threadWhat it involves
Rebuild the spectral pipeline Extract time-frequency features from our 67 participants and run them through the same classifiers. The comparison has never been done on this dataset, and nothing else on this list would settle as much.
Test the phase question A spectrogram discards phase. Our sweep says sign matters. Build features that keep magnitude but scramble phase, and see which side of that the advantage falls on.
Try more electrodes Ten of 256 were chosen for even scalp coverage, following an earlier study, with no test of whether they are the right ten. Nobody has swept this.
Nail down the montage The scalp positions in chapter 3 come from a montage — the file listing where each electrode physically sits — that may not be the one the data was recorded with. If it is the wrong one, every claim about where the signal is needs revisiting.
These are real, currently-unclaimed threads — not exercises with known answers.

Why the small numbers are the point

It would be easy to read chapter 6 and conclude the project failed. AUC 0.58 is not going to read anyone's mind.

But the goal was never a better decoder. The goal is to work out which choices matter, so the next study can fix them in advance instead of rediscovering them. That is what the sweeps are for: not to find the best number, but to find out which knobs are connected to anything.

And the answer so far — preprocessing barely matters, feature count barely matters, the classifier barely matters, but the representation matters enormously — is genuinely useful. It says stop fiddling with artifact thresholds and go think about what the features represent.

Which is why the modest number is worth more than an impressive one. A small, reliable, honestly-reported effect that tells you where to look next is more use than a large one that turns out to be leakage — and large numbers are easy to produce by accident. Chapter 5's “test on the training trials” button gets to 0.73 in one click, and means nothing.
